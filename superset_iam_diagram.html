Key IAM Principles Illustrated

Mapped Role-Based Access Control (RBAC)

Superset's internal roles (e.g., Alpha, Gamma) are dynamically mapped from the user's corporate AD Group claims contained within the JWT, ensuring privileges are always synchronized with the enterprise directory.

Token Security via PKCE

The Authorization Code Flow is secured using PKCE (Proof Key for Code Exchange) with SHA-256 code signing. This mitigation prevents code interception attacks and is mandatory for public clients like the Flask application, maintaining token integrity.

Attribute-Based Access Control (ABAC)

The Flask application uses a policy engine that examines the JWT's claims (attributes) to determine the user's final Superset role, enabling flexible, policy-driven authorization at the application layer.

Identity and Access Federation

The system uses OIDC (OAuth 2.0) and PingFederate to delegate high-assurance Authentication (including the PingID MFA challenge), centralizing credential management and providing robust, context-aware SSO.
