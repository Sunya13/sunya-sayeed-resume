Key IAM Principles Illustrated

Mapped Role-Based Access Control (RBAC)

Internal application roles (e.g., Viewer, Editor, Admin) are dynamically mapped from the user's IdP Group claims contained within the JWT, ensuring privileges are always synchronized with the external directory.

Token Security via PKCE

The Authorization Code Flow is secured using PKCE (Proof Key for Code Exchange) with SHA-256 code signing. This mitigation prevents code interception attacks and is mandatory for public clients like the Flask application, maintaining token integrity.

Attribute-Based Access Control (ABAC)

The Flask application uses a policy engine that examines the JWT's claims (attributes) to determine the user's final application role, enabling flexible, policy-driven authorization at the application layer.

Identity and Access Federation

The system uses OIDC (OAuth 2.0) and an external IdP to delegate high-assurance Authentication (including the MFA challenge), centralizing credential management and providing robust, context-aware SSO.
