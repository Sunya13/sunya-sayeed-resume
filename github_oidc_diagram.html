IAM Principles Highlighted in this Solution

OpenID Connect (OIDC) Federation

We establish OIDC trust between the Cloud Identity Provider (IdP) and the GitHub OIDC issuer, allowing GitHub to vouch for its own workflow identity, eliminating the need for long-term API keys.

Principle of Least Privilege (PoLP)

Access is granted via an IAM Trust Policy which checks specific claims in the JWT, such as the `repository` and `ref` (branch name), ensuring the token is only usable for its intended purpose and source.

Zero Trust / Key Rotation

The access token obtained is short-lived (typically < 1 hour), meaning there are no persistent deployment credentials to steal or expire, drastically reducing the security blast radius of any token leakage.

Non-Repudiation / Auditability

Each token is tied to the specific GitHub workflow run ID and environment (claims in the JWT), providing a clean, immutable audit trail of exactly which CI/CD job performed the deployment.

Proof: IAM Policies Enforcing PoLP and Federation

Trust Policy (Conditioning Role Assumption)

This policy strictly controls WHO can assume the deployment role, limiting access to your GitHub OIDC provider and specifically the `Sunya13` repositories.

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Federated": "arn:aws:iam::AWS_ACCOUNT_ID:oidc-provider/token.actions.githubusercontent.com"
            },
            "Action": "sts:AssumeRoleWithWebIdentity",
            "Condition": {
                "StringEquals": {
                    "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
                },
                "StringLike": {
                    "token.actions.githubusercontent.com:sub": "repo:Sunya13/*"
                }
            }
        },
        {
            "Effect": "Allow",
            "Principal": {
                "AWS": "arn:aws:iam::AWS_ACCOUNT_ID:user/YOUR_USER_NAME"
            },
            "Action": "sts:AssumeRole"
        }
    ]
}

Permissions Policy (Enforcing Least Privilege)

This policy strictly defines WHAT the assumed role can do, granting only the actions necessary for Terraform to manage CloudFront, WAF, S3, and KMS resources.
